# krangka v1.0.1 Upgrade Reference

Changes from v1.0.0 to v1.0.1. Apply these in order when upgrading.

---

## Dependencies

No `go.mod` dependency changes in v1.0.1. Ensure `go.uber.org/mock` is present for mock generation:

```go
// go.mod
require (
    go.uber.org/mock v0.5.2
)
```

---

## Removed

### 1. `configs/version.go` — Delete entire file

**Before (file to remove):**

```go
package configs

import (
	"os"
	"strings"
)

var version string

// GetVersion returns the application version. If .VERSION does not exist (user
// handles versioning via git tags), returns "dev".
func GetVersion() string {
	if version == "" {
		content, err := os.ReadFile(".VERSION")
		if err != nil {
			version = "dev"
			return version
		}
		version = strings.TrimSpace(strings.TrimRight(string(content), "\n"))
	}
	return version
}
```

**Action:** Delete `configs/version.go`.

**Migration:** If your code imports `configs.GetVersion()` (e.g. for Swagger version, health checks, or build info), remove those usages or replace with your own version source (e.g. `-ldflags` at build time, env var, or `.krangka/.VERSION`).

---

### 2. `.gitlab-ci.yaml` — Delete entire file

**Action:** Delete `.gitlab-ci.yaml` from the project root.

**Migration:** CI configuration is project-specific. krangka no longer ships a default GitLab CI file. Add your own CI config (GitLab, GitHub Actions, etc.) as needed.

## Added

### 1. `//go:generate` directives in outbound port files

Add mockgen directives to these 5 port files so mocks can be generated with `make mock`.

| File | Directive to add |
|------|------------------|
| `internal/core/port/outbound/cache.go` | `//go:generate mockgen -source=cache.go -destination=../../../mocks/outbound/mock_cache.go -package=mocks` |
| `internal/core/port/outbound/dlock.go` | `//go:generate mockgen -source=dlock.go -destination=../../../mocks/outbound/mock_dlock.go -package=mocks` |
| `internal/core/port/outbound/idempotency.go` | `//go:generate mockgen -source=idempotency.go -destination=../../../mocks/outbound/mock_idempotency.go -package=mocks` |
| `internal/core/port/outbound/publisher.go` | `//go:generate mockgen -source=publisher.go -destination=../../../mocks/outbound/mock_publisher.go -package=mocks` |
| `internal/core/port/outbound/subscriber.go` | `//go:generate mockgen -source=subscriber.go -destination=../../../mocks/outbound/mock_subscriber.go -package=mocks` |

**Before (example: `cache.go`):**

```go
package outbound

import (
	"context"
	"time"

	"github.com/redhajuanda/komon/cache"
)
```

**After (example: `cache.go`):**

```go
package outbound

//go:generate mockgen -source=cache.go -destination=../../../mocks/outbound/mock_cache.go -package=mocks

import (
	"context"
	"time"

	"github.com/redhajuanda/komon/cache"
)
```

Apply the same pattern to `dlock.go`, `idempotency.go`, `publisher.go`, and `subscriber.go` (adjust source filename in the directive to match each file).

---

### 2. Generated mock files

After adding the `//go:generate` directives, run:

```bash
make mock
```

This creates:

- `internal/mocks/outbound/mock_cache.go`
- `internal/mocks/outbound/mock_dlock.go`
- `internal/mocks/outbound/mock_idempotency.go`
- `internal/mocks/outbound/mock_publisher.go`
- `internal/mocks/outbound/mock_subscriber.go`

**Note:** Do not hand-edit these files. Regenerate with `make mock` after port changes.

## Modified

### 1. `shared/failure/failure.go` — Error code alignment

Standardize failure codes for the first/example domain to `404001` and `409001`.

**Before:**

```go
var (
	ErrNoteNotFound      = &fail.Failure{Code: "404002", Message: "Note not found", HTTPStatus: 404}
	ErrNoteAlreadyExists = &fail.Failure{Code: "409002", Message: "Note already exists", HTTPStatus: 409}
)
```

**After:**

```go
var (
	ErrNoteNotFound      = &fail.Failure{Code: "404001", Message: "Note not found", HTTPStatus: 404}
	ErrNoteAlreadyExists = &fail.Failure{Code: "409001", Message: "Note already exists", HTTPStatus: 409}
)
```

**Migration:** If your project uses different failure codes (e.g. `404002`, `409002`), update them to match your chosen scheme. The framework uses `404001`/`409001` for the Note example domain.

## Migration notes

1. **Remove `configs/version.go`** and any imports/usages of `configs.GetVersion()`.
2. **Delete `.gitlab-ci.yaml`** if present; add your own CI config if needed.
3. **Add `//go:generate mockgen`** to the 5 outbound port files listed above.
4. **Run `make mock`** to generate the new mocks.
5. **Update failure codes** in `shared/failure/failure.go` if you use the Note example.
6. **Update `.krangka/.VERSION`** to `v1.0.1` after applying all changes.
7. **Verify build:** `go build ./...` and `make test`.
